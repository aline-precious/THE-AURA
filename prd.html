Product Requirements Document · v1.0

AttachSecure PRD

A next-generation Attachment Style & Relationship Growth Platform

1. Product Vision

AttachSecure moves beyond a personality quiz to become a living growth platform. It helps individuals and couples transition from insecure attachment styles toward Earned Security through data-tracking, AI coaching, and professional integration. The core premise: security is not a trait you have — it is a capacity you develop.

2. Expanded Feature Set

2.1 The Dynamic Analyzer

Maps interaction patterns between two attachment styles. Not a compatibility score — a lens for understanding the mechanics of relational friction.

  • Anxious-Avoidant Trap: The anxious partner's bids for closeness trigger the avoidant partner's withdrawal — amplifying the anxious partner's fear and pursuit. Both parties respond rationally to their own nervous systems, but the cycle feeds itself. The Analyzer names this clearly and offers interruption strategies for each partner.
  • Secure-Anxious: A stabilising pairing with co-dependency risk. The Analyzer warns when the secure partner functions as an external regulator rather than a model.
  • Disorganized-Secure: Potentially the most healing configuration. The Analyzer equips the secure partner with specific boundary-holding tools.
  • All eight pairings are documented with: (a) core dynamic, (b) typical pain point, (c) growth gift, and (d) communication strategies for each role.

2.2 AI Communication Coach

Takes a message the user is about to send and reframes it to reduce defensiveness and invite connection.

  • User inputs their attachment style (auto-populated) and types the message.
  • System applies a rule-based translation layer trained on attachment theory and Nonviolent Communication (NVC) principles.
  • Output: original message, translated message, and a one-line note explaining what the original might have triggered.
  • Anxious translations: separate accusation from need, add explicit requests, remove catastrophising language.
  • Avoidant translations: add return signal to withdrawal, insert one small emotional disclosure, replace shutdown language.
  • Disorganized translations: resolve approach-avoidance mixed signals into a single clear request.
  • v2 roadmap: Two-way mode where both partners translate simultaneously via LLM API integration.

2.3 Progress Dashboard

Tracks movement toward Earned Security using:

  • Security Score (0–100): Weighted composite of quiz answer distribution, check-in mood trends, and streak consistency.
  • Mood Trend: 7-day and 30-day rolling averages from daily check-ins. Visualised as an animated bar chart.
  • Trigger Frequency: How often high-stress events are logged and whether trigger types shift over time.
  • Check-in Streak: Consecutive daily engagement. Research supports consistent self-reflection as intrinsically therapeutic.
  • Coach Engagement: Number of messages translated, indicating active practice of secure communication.

4. Daily Micro-Interactions

01 · The Morning Mirror

Format: Single-screen 60-second prompt, personalised to attachment style.

Mechanic: Each morning, one reflection question tied to the user's style (e.g. for Anxious: "Before you check your messages today, name one thing you're proud of in yourself."). Respond with an emoji reaction or a sentence — no pressure.

Why it works: Builds the habit of internal reference before external check — the core skill for moving from anxious to secure.

02 · The Evening Pulse

Format: 1–10 mood slider + optional trigger tag + optional note. Takes 90 seconds.

Mechanic: At a user-set time, a gentle push notification: "How did connection feel today?" The data feeds the progress dashboard. When a score drops to 3 or below, the Trigger Alert activates immediately.

Why it works: Longitudinal mood tracking correlates strongly with therapeutic outcome. The user builds a map of their own patterns over time.

03 · The Repair Prompt

Format: Conditional trigger — appears only when a high-stress check-in is logged or a "conflict" trigger is selected.

Mechanic: After a high-stress entry, the platform surfaces: "Is there something you could say or do in the next 24 hours to move toward connection — even a small thing?" If the user responds, the AI Coach offers to help craft it.

Why it works: Repair — not the absence of conflict — is the hallmark of secure attachment. This micro-interaction builds the repair reflex.

5. Trigger Alert Technical Logic

A Trigger Alert fires when:

  • Mood check-in score ≤ 3 (out of 10)
  • User selects a trigger tag signalling relational distress
  • Security score drops more than 10 points in a 7-day window

UI behaviour on alert

  • Full-width amber banner appears at the top of the current page with a style-specific grounding message.
  • Banner is non-dismissible for 60 seconds — prevents immediate avoidance.
  • After the lock, a dismiss button appears alongside two action options: "Open the Coach" and "Write it out" (private journal).
  • If the user has a therapist linked, a secondary notification is queued to the therapist's inbox (opt-in, mutual consent required).
  • For Disorganized users, the alert includes a brief grounding exercise (5-4-3-2-1 sensory method) before any text.

Data handling

  • Trigger events logged with timestamp and style-specific response provided.
  • Aggregate trigger data feeds the dashboard's Trigger Frequency chart.
  • No trigger data shared with third parties. Therapist access requires explicit user consent per event — not blanket permission.

6. Technical Architecture

  • Backend: Go — stateless, performant, deployed as Vercel serverless function via api/index.go.
  • State: Server-side sessions (gorilla/sessions with cookie store). Migrate to PostgreSQL for persistent user accounts in v2.
  • Frontend: Server-rendered HTML templates (Go html/template) with vanilla JS. No framework dependency — fast initial load, zero hydration.
  • AI Coach: Rule-based translation engine (v1) → LLM integration via Anthropic/OpenAI API (v2, with attachment-theory system prompt).
  • Deployment: Vercel (Go serverless) via vercel.json. Static assets served from /static/.
  • Auth (v2): Email magic-link authentication. No password storage. OAuth for therapist role verification.

7. Success Metrics

  • 30-day retention rate ≥ 40% — driven by daily check-in streak mechanics
  • Average security score increase of +8 points over 4 weeks of consistent engagement
  • Coach usage: ≥ 3 messages translated per active user per month
  • Therapist NPS ≥ 50 from licensed users
  • Trigger alert follow-through rate ≥ 60% — user takes an action after the alert fires
